4.3 Diagrammes de séquence

Séquence 1 — Création d'un projet

📌 Séquence visuelle — Création d'un projet
sequenceDiagram actor User as Utilisateur participant Vue as Vue (Create.vue) participant Inertia as Inertia.js participant Ctrl as ProjectController participant Policy as ProjectPolicy participant DB as Base de données participant Audit as ActivityLog User->>Vue: Remplit le formulaire & clique "Créer" Vue->>Inertia: form.post(route('projects.store')) Inertia->>Ctrl: Requête POST /projects Ctrl->>Policy: authorize('create', Project) alt Non autorisé Policy-->>Ctrl: ❌ Erreur 403 (Aborted) Ctrl-->>Inertia: Retour 403 Inertia-->>User: Affiche message d'erreur (403) else Autorisé Policy-->>Ctrl: ✅ Autorisé Ctrl->>Ctrl: Validation des données (Request) alt Erreurs de validation Ctrl-->>Inertia: Redirection back avec erreurs (422) Inertia-->>User: Affiche les erreurs du formulaire else Validation OK Ctrl->>DB: Project::create([...]) DB-->>Ctrl: Instance Project créée Ctrl->>Audit: Enregistre l'activité ("Projet créé") Ctrl-->>Inertia: redirect()->route('projects.show', $id) Inertia-->>User: Affiche la page de détail du projet end end
Acteur → Composant Action / Message Retour
Utilisateur → Vue Create.vue Saisit le formulaire et clique "Créer" —
Vue → Inertia.js form.post(route('projects.store')) —
Inertia → ProjectController POST /projects —
Controller → ProjectPolicy authorize('create') ✅ Autorisé
Controller → Validation $request->validate([...]) ✅ ou erreurs 422
Controller → Project (Model) Project::create([...]) Instance créée
Controller → ActivityLog activity()->log('Projet créé') —
Controller → Inertia redirect()->route('projects.show') ✅ Page détail

Séquence 2 — Génération d'un rapport mensuel

📌 Séquence visuelle — Génération d'un rapport mensuel
sequenceDiagram actor Admin as Administrateur participant Vue as Vue (Reports/Index.vue) participant Ctrl as ReportController participant DB as Base de données participant DomPDF as DomPDF / CSV Service participant Storage as Stockage (Storage) participant Model as MonthlyReport (Model) Admin->>Vue: Sélectionne mois/année/format & clique "Générer" Vue->>Ctrl: Requête POST /reports/generate Ctrl->>DB: Récupère projets actifs du mois DB-->>Ctrl: Collection de projets Ctrl->>DB: Récupère dépenses du mois + stats N-1 DB-->>Ctrl: Collection de dépenses & agrégations Ctrl->>DB: Vérifie rapport existant DB-->>Ctrl: Rapport existant ou null alt Rapport déjà existant Ctrl->>Storage: Supprime l'ancien fichier physique end Ctrl->>DomPDF: Génère le fichier binaire (PDF/CSV) DomPDF-->>Ctrl: Fichier généré Ctrl->>Storage: Stocke le fichier dans 'reports/rapport_*.pdf' Storage-->>Ctrl: Retourne le file_path Ctrl->>Model: MonthlyReport::create([...]) Model-->>Ctrl: Enregistrement sauvegardé Ctrl-->>Vue: Redirection avec message flash de succès Vue-->>Admin: Affiche la liste des rapports mise à jour
Acteur → Composant Action / Message Retour
Admin → Reports/Index.vue Sélectionne mois, année, format (PDF/CSV) —
Vue → ReportController POST /reports/generate —
Controller → Base de données Récupère projets actifs du mois Collection
Controller → Base de données Récupère dépenses du mois + stats N-1 Collection
Controller → Base de données Vérifie rapport existant Existant ou null
Controller → Storage Supprime ancien fichier si existant —
Controller → DomPDF / CSV Génère le fichier rapport Fichier binaire
Controller → Storage (public) Stocke dans reports/rapport_*.pdf Chemin fichier
Controller → MonthlyReport MonthlyReport::create([...]) —
Controller → Inertia Redirige vers /reports avec succès ✅ Liste rapports

Séquence 3 — Saisie d'une dépense avec justificatif

📌 Séquence visuelle — Saisie d'une dépense avec justificatif
sequenceDiagram actor Chef as Chef de Projet participant Vue as Vue (Expenses/Create.vue) participant Ctrl as ExpenseController participant ExpPol as ExpensePolicy participant ProPol as ProjectPolicy participant Storage as Stockage (Storage) participant DB as Base de données participant Audit as ActivityLog Chef->>Vue: Renseigne la dépense & charge le justificatif Vue->>Ctrl: Requête POST /expenses (multipart/form-data) Ctrl->>ExpPol: authorize('create', Expense) ExpPol-->>Ctrl: ✅ Autorisé Ctrl->>ProPol: authorize('update', $project) (vérifie l'ownership) alt Projet ne lui appartient pas ProPol-->>Ctrl: ❌ Erreur 403 Ctrl-->>Vue: Retour 403 else Projet lui appartient ProPol-->>Ctrl: ✅ Autorisé Ctrl->>Ctrl: Validation des champs & fichier alt Fichier présent Ctrl->>Storage: file->store('receipts', 'public') Storage-->>Ctrl: receipt_path end Ctrl->>DB: Expense::create([...]) DB-->>Ctrl: Instance Expense créée Ctrl->>Audit: Enregistre l'activité ("Dépense enregistrée") Ctrl-->>Vue: Redirige vers la vue détail du projet Vue-->>Chef: Affiche la page du projet avec la dépense ajoutée end
Acteur → Composant Action / Message Retour
Chef de projet → Expenses/Create.vue Remplit formulaire + upload justificatif —
Vue → ExpenseController POST /expenses (multipart/form-data) —
Controller → ExpensePolicy authorize('create', Expense) ✅
Controller → ProjectPolicy authorize('update', $project) ✅ Si ownership OK
Controller → Storage file->store('receipts', 'public') receipt_path
Controller → Expense (Model) Expense::create([...]) Instance créée
Controller → ActivityLog activity()->log('Dépense enregistrée') —
Controller → Inertia Redirige vers projects.show ✅ Page projet
